Introduction #

All parts of the system are using environment variables. Environment specific configuration is parsed to process.env variables. All applications are using dotenv and dotenv-expand to set environment variables.

Each application / package loads .env file in respective directory. There is example file (.env-template), which can be used as template.

Copy the .env-template file and modify the .env file as needed.

cp .env-template .env

API Server #

Configuration files are stored in src/config directory of application or package.

Configuration is structured to following sections:

  • Application
  • Access Control List (ACL)
  • Cache
  • Database
  • Filesystem
  • Mail
  • Logging
  • Queue
  • Security
  • Services
  • Session

Admin client #

Admin client is Vue.js based application and scaffolded with vue-cli. Docs outline specific settings for deployment of the application. But feel free to dive in to vue-cli docs if any further modifications are required.
